# Epic & User Stories — Tax Calculator Modernization

---

## Epic

| Field            | Details |
|------------------|---------|
| **Title**        | Modernise Tax Calculator |
| **Description**  | The Tax Calculator currently runs as a manually deployed static application and lacks a robust pipeline in place. This epic focuses on modernizing the application to deploy it on IBM Cloud. The web application will not just be simply lifted and shifted to cloud. The application will be containerized using a Docker container. The deployment will be managed through a pipeline, which will also ensure that all unit tests pass before deployment. |

---

## User Stories

### Story 1 — Containerize the Application

| Field            | Details |
|------------------|---------|
| **Title**        | Containerize the Tax Calculator application |
| **Description**  | To ensure that each unit of code performs as expected, run unit tests. And if the unit tests pass, deploy the application to a Docker container. |
| **Acceptance Criteria** | 1. A `Dockerfile` exists in the project root. 2. The image builds successfully with `docker build`. 3. The container serves the application on port 5000. 4. Unit tests pass before the image is built. |
| **Story Points** | 3 |
| **Priority**     | High |

### Story 2 — Deploy on IBM Cloud

| Field            | Details |
|------------------|---------|
| **Title**        | Deploy the containerized application on IBM Cloud |
| **Description**  | Instead of hosting on self-managed or in-house virtual machines, deploy the application on IBM Cloud, using IBM Cloud Code Engine. |
| **Acceptance Criteria** | 1. The container image is pushed to IBM Cloud Container Registry. 2. An IBM Code Engine project is configured. 3. The application is accessible via a public URL generated by Code Engine. |
| **Story Points** | 5 |
| **Priority**     | High |

### Story 3 — Create a Pipeline for Packaging and Deploying

| Field            | Details |
|------------------|---------|
| **Title**        | Create a Tekton pipeline for build, test, and deploy |
| **Description**  | The current process of packaging and deploying involves too many manual steps. Use a Tekton pipeline to automate the build, test, and packaging stages of the application. |
| **Acceptance Criteria** | 1. A Tekton Pipeline with tasks for clone, lint, test, build-image, and deploy. 2. Pipeline runs triggered on push to the main branch. 3. Deployment proceeds only when all tests pass. |
| **Story Points** | 8 |
| **Priority**     | High |

### Story 4 — Externalize Tax Bracket Configuration

| Field            | Details |
|------------------|---------|
| **Title**        | Replace hard-coded tax brackets with external configuration |
| **Description**  | Move the tax bracket data out of the source code and into a JSON configuration file so that brackets can be updated without code changes. |
| **Acceptance Criteria** | 1. Tax brackets are read from `config.json` at runtime. 2. Changing the file changes calculation results without redeploying code. 3. Existing unit tests still pass. |
| **Story Points** | 2 |
| **Priority**     | Medium |
